# OpenKlip - Roadmap

## Status

OpenKlip is a lean, local-first **agent-native video toolchain**: external agents drive the edit loop via CLI; humans review in the browser. Both read/write the same `project.json` on disk.

**Current release:** v0.41.1.0 (2026-07-06 in `VERSION`, `package.json`, and `CHANGELOG.md`; merged to `main` at `cea7939`, PR #78). Working local editor: cut → captions (five style presets: boxed, clean, karaoke, bold-caps, minimal; optional vertical safe-area inset on export) → cleanup review (filler/dead-air candidates; brief **Always cut** / **Never cut** lists) → b-roll → vignette → push-in zoom → titles → grade/LUT → rich graphics → json-render graphics (`product-announcement`, `map-motion`) → music placement → export audio quality (VAD snap, seam crossfades, ducking, loudness, de-essing) → export (compression presets, output frame rate, named platform presets for YouTube/YouTube 4K/X/LinkedIn/Shorts, MP4/GIF format with a size/duration cap on GIF that CLI/MCP/route/action/GUI can override up to a 1920px hard ceiling, and file/clipboard destination); browser project creation (upload, **Import folder…**, **Import from URL**, multi-file drop, or **Blank canvas**); transcript phrase search with batch cuts and restore; real export settings (compression presets + output frame rate + platform presets); append-only action history covering every user-facing mutation, with pre-mutation snapshots, task-level revert (`openklip revert`, MCP `revert`, GUI History panel with its own actor/action/task filter UI), and agent-queryable history/task lookups (`openklip history`/`tasks`, MCP `history_list`/`task_list`, both filterable by actor and both validating `--actor` against a fixed enum); project brief with CLI/MCP/GUI parity; agent tasks with visible progress and cancel; `make-draft` and `revise-draft` one-prompt playbooks; cinema player with live graphics/titles/captions overlays and its own cut-aware `CutScheduler` matching the inline preview; resizable right chat sidebar with Claude MCP edits; right-side Config shell with color temperature controls and a **Graphics** picker (template packs, hover previews, beat-snapped placement, cut-seam transitions); smaller-screen Chat and Config overlays; asset cards (`openklip analyze` + **Describe assets**); b-roll suggest (`openklip broll-suggest`, MCP `broll_suggest`); skills slash menu with an agent-native skill index and `load_skill` tool; MCP agent tools (89 tools); 44 registry actions; edit templates; native HTML/CSS graphics templates (pixel-faithful headless-Chrome export to ProRes 4444 alpha), including an 8-template motion text pack, five transition templates, 29 bundled shader templates, project-local graphics overrides, and blank-canvas motion projects; `graphics/AUTHORING.md` for folder-drop template authoring; graphic keyframe animation (declarative opacity/scale/x-y keyframes on graphic overlays with seven easings including spring/backOut/anticipate, timeline diamond markers + inspector editing, `graphic-set`/CLI/query parity, one frame-pure code path for preview and export); written rationale notes on cuts/overlays; phrase-anchored cues that re-snap after a re-cut; multi-take assembly with a GUI browse-and-assemble-and-upload takes panel; `openklip cuts-snap` CLI; music timeline drag-trim and preview gain 0-2; project-local graphics upload in Config; `brief_audit` on export skills (make-short, make-highlights, talking-head, motion-*); `bun run agent-smoke-audit` in CI; browser integration CI (History transcript diff + json-render timeline + mobile overlays, `OPENKLIP_INTEGRATION=1`); 1888 tests (1885 pass, 3 skip without integration env); MIT.

Preview cuts now play a decorative `glimm` WebGL sweep at each auto-advance cut boundary (`web/scheduler.ts`'s `onCutBoundary`, `web/components/cut-transition-sweep.tsx`), matching `project.look.transition`'s type (crossfade: a bright azure dissolve-like flash; dip: a near-black sweep). It respects `prefers-reduced-motion` and degrades to no visual effect when WebGL is unavailable; it does not fire on manual scrub/seek, and does not pixel-match the export side's ffmpeg `xfade`/fade-pair transition, just evokes the same two transition types. `CinemaPlayer` now drives its own `CutScheduler` instance (v0.32.0.0) and shares this sweep; previously (through v0.31.0.0) it had no cut-skip logic at all and played the raw, uncut proxy start to finish despite shipping as a `[x]`-marked complete feature (see Known Limitations for the fix and its residual gaps). Exported MP4s can use VAD-snapped audio seam crossfades when cut snap is enabled, and crossfade/dip-to-black transitions for voice-only exports (no b-roll/stills/music/rich graphics).

## Architecture & Key Decisions

- Files-on-disk `project.json` EDL; no database; GUI and terminal agent edit the same file (parity is the core agent-native bet).
- The agent is Claude Code / Codex / Cursor / Grok via the user's own subscription CLI; OpenKlip ships **no** bundled LLM, no API keys, no cloud.
- Whisper via Transformers.js, picked over native whisper.cpp (no cmake / build toolchain needed).
- ffmpeg via `ffmpeg-static`: GPL binary invoked as a subprocess, so it does **not** bind OpenKlip's MIT license.
- Remotion deliberately **not** used (commercial license at 4+ employees); preview is a native `<video>` scheduler, export is ffmpeg `filter_complex`.
- All-intra 720p proxy for instant seeks; 8-bit `yuv420p` everywhere; export re-encodes from original source on the same 48 kHz sample grid as preview.
- ffmpeg `crop` can't vary size per-frame → animated zoom uses `zoompan` (per-frame `z`); static effects use `split` + `overlay`.
- Server-side `project.json` mutations serialize per-slug in-process (`withProjectLock` / `mutateProject`) plus a cross-process advisory lock on `project.json` (`src/project-file-lock.ts`). Unrelated files (chats, tasks, brief) still have separate locks.

## Known Limitations

Single list of current gaps (code is truth). README and release notes point here for detail.

### Export & media

- 4K export re-decodes the whole source when segment mode is off (slow for sparse cuts on long sources). Segment seeking (v0.28.0.0) helps voice-only exports when kept duration is under half the source; b-roll and rich graphics still force full-source decode, while music beds and still overlays can ride segment mode.
- GIF export is capped (v0.32.0.0, `src/exporter.ts`): the second full ffmpeg pass (palettegen/paletteuse) that converts the already-rendered mp4 clamps width to 960px and frame rate to 15fps (`GIF_MAX_WIDTH_PX`, `GIF_MAX_FPS`, `clampGifDimensions`, aspect-preserving), and a kept duration over 5 minutes (`GIF_MAX_DURATION_SEC`) throws before any ffmpeg work runs rather than producing a huge file. The mp4 pipeline is unaffected. A width override now exists (v0.33.0.0): CLI `--gif-max-width <px>`, the MCP `export` tool's `gifMaxWidth` input, the export route, and the `exportProject` server action can raise the 960px default up to a hard ceiling of `GIF_MAX_WIDTH_OVERRIDE_CEILING_PX` (1920px), enforced inside `clampGifDimensions` itself so no surface can bypass it. A GUI control now exists too (v0.33.0.1, `web/components/export-dialog.tsx`/`export-options-form.tsx`): a numeric input next to the "GIF exports have no audio." hint when GIF format is selected, clamped client-side to `[1, GIF_MAX_WIDTH_CEILING_PX]` (1920) and only submitted when it differs from the 960px default. Frame rate (`GIF_MAX_FPS`) and duration (`GIF_MAX_DURATION_SEC`) still stay fixed on every surface with no override.
- Clipboard destination copies only the exported file's absolute path as text (`navigator.clipboard.writeText` in `web/app.tsx`); the browser Clipboard API cannot hold binary video, so no video bytes are ever copied, and the copy never reaches the server. Destination selection (file/clipboard) is GUI-only by design, not a CLI/MCP gap: Clipboard is a client-side browser API call with no server-side or CLI/MCP concept to wire into. Export format (`mp4`/`gif`) now has full CLI/MCP/route/action/GUI parity (closed in v0.29.0.1).
- The export dialog's size/time estimate is approximate; GIF format uses a separate clamped-width/fps profile and shows a distinct note.
- Preview cut transitions (v0.31.0.0, `web/lib/cut-transition-sweep.ts`, `web/components/cut-transition-sweep.tsx`) are a decorative `glimm` WebGL sweep, not a pixel-accurate preview of the export side's ffmpeg `xfade`/fade-pair transition: it evokes crossfade (bright azure dissolve) versus dip (near-black sweep) but the visual language is different from what actually renders. It only fires on playback auto-advance (`web/scheduler.ts`'s `jumpToRange`), never on a manual scrub/seek. Exported MP4s support crossfade and dip-to-black transitions for voice-only exports (no b-roll/stills/music/rich graphics); when overlays are present, too many ranges are kept, or fewer than two ranges survive, the export falls back to a hard cut, but this is no longer silent: `ExportResult.transition` (`src/exporter.ts`) reports `{ type, applied, reason }`, the CLI export summary prints a `transition <type> requested but not applied (<reason>)` fragment (or confirms it applied), and the GUI surfaces an info toast explaining the fallback in plain language.
- **CinemaPlayer cut-skip bug, found and fixed (v0.32.0.0):** the fullscreen cinema preview (`web/components/cinema-player.tsx`) had zero cut-skip logic despite the cinema player shipping as a `[x]`-marked complete feature; it played the raw, uncut proxy file start to finish, showing every deleted word or range. Fixed by giving `CinemaPlayer` its own `CutScheduler` instance (the same class the inline preview uses, not a second implementation), driven by `getRanges`/`getTransition` props from `web/app.tsx` (the same `rangesRef`/`projectRef` the inline preview already reads, no duplicate computation), which also picked up the Glimm cut-transition sweep. Fixing this surfaced a second bug in the same code path: the graphics/titles/captions overlay stack was keyed against raw source time instead of post-cut-jump source time, so overlays could show against the wrong frame during cinema playback; both are fixed together. The pure cut-space position math (`outputPositionSec`/`sourceSecForOutputPosition`) moved out of `web/app.tsx` into shared `src/schedulerLogic.ts` so the two preview surfaces can't drift again. `CutScheduler` gained a `.dispose()` method (closes its `AudioContext`) since CinemaPlayer's video element mounts and unmounts repeatedly (toolbar open/close), unlike the inline preview's, which mounts once; without it, repeated open/close would leak `AudioContext`s. `tests/cut-scheduler-dom.test.ts` exercises `.dispose()`, manual `seek()` vs `onCutBoundary`, and overlay-time sync via `outputPositionSec`.
- B-roll supports cover, PiP, and split display modes plus b-roll audio ducking; title cards support lower, center, hero, quote, divider, and callout positions.
- HyperFrames post-export (`openklip package`) is opt-in: requires separate `hyperframes` npm install and Chrome; not bundled.
- `product-announcement` and `map-motion` are the json-render graphic catalogs today; `json-graphic-add` / `json-graphic-set` accept only those catalog ids. An invalid json-render spec now shows an "Invalid graphic spec" card in the editor preview and export fails before ffmpeg with a clear error (no silent skip).
- **Map motion** (v0.41.1.0): headless export fetches Carto basemap tiles over the network each frame; offline CI, firewalls, or CDN outages fail after load/idle timeouts (no infinite hang). Image-filter shader previews in the Graphics picker still need a registered `assetId` param. Multiple rich/json-render overlays (including several map-motion clips) still render in parallel (`Promise.all` in `src/exporter.ts`), which can spike RAM and network with no concurrency cap.
- Rich shader templates (29 bundled `graphics/shader-*` ids, `@paper-design/shaders@0.0.77`) require WebGL2 in both browser preview and headless export. Five image-filter shaders load a registered still or image b-roll via `assetId` (heatmap uses CPU preprocessing through `@paper-design/shaders`'s `toProcessedHeatmap` before render). There is no non-WebGL fallback renderer yet.
- **Blank canvas** (v0.41.0.0): motion projects have no camera source; export spans the full timeline when the transcript is empty. `openklip verify` short-circuits until an export exists.
- **Beat-snapped graphics** (v0.41.0.0): beat placement needs a BPM on the chosen music asset (`music_bpm` / GUI **Detect BPM**). Without a music asset, placement falls back to seconds-based spans.
- **`graphic-add-cuts` transition seams** (v0.41.0.0): places rich `transition-*` overlays centered on kept-range cut boundaries only. Export-side ffmpeg `xfade`/dip transitions remain voice-only (see preview cut transitions above).
- **Project-local graphics** (v0.41.0.0): templates under `projects/<slug>/graphics/` override bundled ids on collision. Discovery is folder-based; the Config **Graphics** picker also accepts a template id plus `manifest.json` and `composition.html` upload (POST `app/api/projects/[slug]/graphics/route.ts`).
- Motion text pack (v0.39.0.0, `graphics/motion-*`): `motion-roll-number`'s `rollNumber` effect does whole-number interpolation only, no per-digit slot-machine roll animation and no `Intl`/locale number formatting, by design (plain integer plus prefix/suffix params). Manifest params include `inDurFrames` and `staggerFrames` (wired through `data-timing-bind` on export and preview). `motion-blur-reveal`'s per-character CSS filters can get heavy in the live preview for very long strings (export is unaffected, since it renders frame-by-frame through headless Chrome rather than live CSS transitions). `motion-shimmer` depends on its own template CSS `background-clip: text`; the shared runtime only drives `background-position`, so a hand-authored template that omits the clip declaration will not shimmer.
- json-render timeline select/trim, History transcript diff, and mobile Chat/Config overlays have browser integration tests (`tests/json-graphic-browser.test.ts`, `tests/transcript-diff-browser.test.ts`, `tests/mobile-overlays-browser.test.ts`); they skip in default `bun test` unless `OPENKLIP_INTEGRATION=1` and Chrome are present, and CI runs them in the `integration` job.
- Export platform presets include a `shorts` destination (9:16, source fps, 1920 height cap); landscape presets (`youtube`, `youtube-4k`, `x`, `linkedin`) stay 16:9-honest. Reframe crop (manual, scene, vision) is supported on macOS and via sceneLog; continuous subject tracking across time is not.
- A platform preset's `fps` is a hard pin, not a cap: exporting 24/25fps footage with the `x` or `linkedin` preset (pinned to 30fps) retimes up to 30fps, which can duplicate frames and judder. This matches how an explicit `--fps` has always behaved; pass `--fps 24` (CLI) or set frame rate to Source (GUI) after picking the platform to opt out.
- CLI `--loudness` accepts `-30..-10` or `off` to skip normalization for this export even when a platform preset would apply one.

### Audio & music

- Preview music gain now honors export's 0-2 range via a Web Audio gain node (`web/lib/music-preview-audio.ts`); falls back to element volume when Web Audio is unavailable.
- All export audio quality processing (ducking, loudness normalization, voice highpass, seam crossfades) is export-only; preview audio is always unprocessed.
- Single-pass loudnorm remains the default (`loudness.mode: single`); two-pass is opt-in for exact targeting.
- Seam crossfades borrow up to half their duration from already-removed audio on each side of a cut; very short kept ranges clamp the crossfade down to a duration-preserving butt join instead.
- The Cleanup panel now shows registered dead-air spans with per-span remove buttons (`dead-air-rm` GUI surface). The GUI History panel's revert remains available for bulk undo.
- `src/audio-analysis.ts` reads the whole ingest-time PCM (`working/audio16k.f32`) into memory for silence detection; fine for typical footage, unbounded for hours-long sources.
- Noise reduction and de-essing both ship on the voice bus: light `afftdn` noise reduction, and (v0.32.0.0) ffmpeg's bundled `deesser` filter (`project.audio.deEsser`, `enabled`/`intensity` 0-1 default 0.5), applied after highpass and noise reduction (highpass -> afftdn -> deesser) on both the plain and VAD-snap-crossfade voice paths (`src/exporter.ts`, `src/export-segments.ts`). `openklip audio <slug> --deess on|off --deess-intensity <0-1>`, MCP `audio` action, and a GUI Audio section toggle+slider. Only `intensity` (the filter's `i` option) is exposed; the filter's frequency (`f`) and output-mode (`s`) parameters are hardcoded to ffmpeg's own defaults, not user-configurable.

### Editing & transcript

- Whisper word timestamps are ~100 ms loose. VAD snap-to-silence (`cuts.snap`, GUI Config panel Audio section, MCP `cuts-snap` action, and `openklip cuts-snap <slug>`) tightens cut boundaries onto detected silence when enabled, and seam crossfades reuse a few ms of removed audio to declick the join; snap resolution is bounded by the 20ms audio-analysis window (not sample-accurate).
- Cleanup candidates (filler words, dead air) split "safe" (fine to auto-apply) from "review" (ambiguous fillers, multi-word phrases, anything within 0.3s of an overlay); only "safe" candidates are meant for unattended auto-apply, review candidates need a human or explicit agent judgment call. Brief `Always cut:` / `Never cut:` lines and optional `project.cuts.cleanupPhrases` extend the filler list (`openklip cleanup`, MCP `cleanup_report`, GUI Cleanup panel).
- Phrase-anchored overlays re-snap to their anchor on every cut: a manual `*-set` span on a phrase-placed overlay can be re-moved by the next word deletion (place a plain `*-add` overlay to pin a span). A deleted anchor phrase leaves the overlay `stale` with its last good span until the words are restored.
- Multi-take assembly now has a GUI browse-and-assemble panel (v0.33.0.0, Config sidebar Takes section between Highlights and Music, `web/components/takes-panel.tsx`): browse ingested takes, click-select a word range per take, build a multi-segment selection across takes, and assemble. Ingesting a new take also has GUI upload now (v0.34.0.0): an "Add take" file-picker control POSTs to `app/api/projects/[slug]/takes/route.ts` and polls the same background-job infrastructure the whole-project upload uses, with phase messages during ingest (`ingestTake` reports probe/proxy/audio/transcribe). Uploading a file whose id/label collides with an existing take silently overwrites it, intentionally matching the CLI `take-add` behavior rather than adding a GUI-only confirmation dialog. Each take is ingested with the full ffmpeg + Whisper path and parked in `takes/<id>/`; `assemble` writes a new single-source project and needs `--force` to overwrite an existing edit.
- Phrase search (UI and CLI/MCP `transcript grep`/`phrase`) can miss a cut phrase when Whisper's tokenization of the spoken audio differs from the phrase text a user or agent types; the search returns an honest empty result rather than a fuzzy guess.
- The transcript contentEditable does not restore a cut word when you type its text back into place (behavior change, 2026-07-02): deleted words stay visible, struck through, in the editable text, so a word's text merely reappearing is not treated as evidence of restoration and the word keeps its `deleted` flag. Restoring a cut word is always an explicit action: the timeline toggle, transcript search Restore/Restore all, Cleanup revert, or `openklip revert`.
- **Edit provenance (optional UI)**: action history always records `authorId`, `model`, and `agentSurface` when env vars are set. The GUI shows attribution only when Settings → Appearance → **Show edit attribution** is on (default off): transcript hover, History author filter, **View in history**, and title/zoom/still overlay inspectors (b-roll paint-order list already showed author suffix). Overlays stamp `authoredBy` on add/set. Legacy history entries and words edited before v0.40.0.0 may lack `authorId` / `authoredBy`. `tests/transcript-diff-browser.test.ts` and `tests/json-graphic-browser.test.ts` skip unless `OPENKLIP_INTEGRATION=1` and Chrome is present; each bootstraps its own fixture project and dev server on a free port. CI runs them via the `integration` job.
- Per-platform caption safe areas are preview guides by default; optional `captions.insetPlatform` (`openklip captions-inset`, MCP `captions-inset`, `agent-make-short`) lifts burned captions on vertical exports using the same inset fractions as the preview guides. Split-screen vertical export layout is on `project.export.layout` (distinct from b-roll `display: split`).

### Workspace & platform

- Workspace folder picker is **macOS-only**; Linux/Windows can set a projects root via `OPENKLIP_PROJECTS_ROOT`, `POST /api/workspace` with `{ action: "set", path }`, or the path field in the New project dialog. Empty landing accepts a single video, multi-file folder drop, or URL import (yt-dlp on PATH); the largest supported video becomes the primary ingest and sidecars land in `assets/`.
- Ingester plugins: folder batch and URL download are wired in the GUI New project dialog (`POST /api/projects/folder`, `POST /api/projects/url`); other manifests remain CLI discovery only (`openklip ingesters`).
- Browser uploads stream to disk with size caps (12 GB project, 4 GB asset); URL downloads use yt-dlp locally and inherit the same ingest path.
- `actions.jsonl` has no rotation. History reads with a `--limit` / API cap now tail-read the log instead of parsing the whole file when a limit is passed (`readActionLog` in `src/action-log.ts`); unbounded reads still load the full file.
- When source persist fails after a successful ingest, the job finishes as `partial`: the project opens with a warning toast and exports fall back to the proxy until the source is fixed manually.

### Agent & chat

- OpenKlip ships **no bundled LLM** for the core edit loop; external agent, MCP, or manual CLI drives mutations.
- **Claude chat edits** load the openklip MCP server and mutate `project.json` in-process. Other agents answer via CLI or return read-only hints; skills slash menu still routes to suggested command sequences.
- **Find filler** and **Describe assets** shell out to the selected agent CLI; needs that CLI on PATH and signed in. Cursor requires one-time `cursor-agent login`.
- Agent tasks (`working/tasks.json`) are cross-process safe (an advisory lockfile serializes the Next server and the spawned MCP process, so a stale step report cannot resurrect a cancelled task) and cancel kills the run's whole process group. On server start, any task still marked `running` or `pending` is finalized as `failed` with a restart message, since the in-process run registry does not survive a dev-server restart.

### Architecture & parity

- GUI server actions do not dispatch through `runAction()`: CLI uses `src/registry.ts`; GUI uses `app/actions.ts` + `projectMutations` (same `project.json`, separate code paths).
- Project write locks are in-process per slug plus a cross-process advisory lock on `project.json` (`src/project-file-lock.ts` inside `mutateProject`). Two concurrent processes writing different files (e.g. chats vs project) can still race unrelated paths.
- Action history now covers every user-facing mutation: registry actions, GUI direct-save paths, asset registration/deletion, `template set`, `brand`/`ingest --brand`, multi-take `assemble` (through `mutateProject`, revision counter preserved), background folder-sync prune (`asset-prune`, actor `system`), and brief saves from CLI/GUI/MCP (one best-effort `brief-set` entry, no EDL revision bump). Revert (`openklip revert`, MCP `revert`, GUI History panel) undoes to a revision, a task's start, or the last edit. Agents can filter history by task id, action name, or actor (`openklip history`/MCP `history_list`, actor filter added v0.32.0.0), and the GUI History panel now has its own actor/action/task filter UI (v0.33.0.0, `HistoryFilterControls` in `web/components/history-panel.tsx`, AND-semantics, client-side over the already-fetched entries). Revert restores `project.json` only: `brief.md`, chats, tasks, asset files, and derived media (proxy, extracted audio, transcript) are not restored, snapshots are capped at the newest 100 revisions, and a revert cannot cross a multi-take `assemble` boundary (refused, since the on-disk media no longer matches). A CLI/MCP revert while the editor is open in a browser leaves the client stale until the next reload, the same pre-existing class as any external edit; GUI-initiated reverts reseed the open editor's state directly.
- **Fixed (v0.33.0.1):** the GUI History panel's "history truncated" warning (`canRevertGroup`'s `truncated` check against `HISTORY_PAGE_LIMIT`, 200) previously compared the FILTERED entry count against the 200-entry page limit; when an actor/action/task filter narrowed the visible view below 200 entries, a truncation warning that should still apply (the raw unfiltered fetch WAS truncated at 200) could go silently missing. `groupTouchesTruncationBoundary` now delegates to a new pure `shouldShowTruncationWarning(rawFetchedCount, limit)`, and `HistoryPanel`/`HistoryList` (`web/components/history-panel.tsx`) thread the true unfiltered `rawEntries` alongside the filtered `entries` so the check always reads the raw fetch, never the displayed subset. The disabled-affordance copy also now varies: it names the active filter as a reason a narrow-looking view doesn't prove the underlying history is short, staying honestly non-committal about whether more matching entries exist beyond the fetched page either way.
- `ProjectSchema` is now `.passthrough()`: unknown top-level keys survive a load/save round-trip instead of being silently dropped by an older build, closing the previous forward-compat gap.
- Revert only touches `project.json`, not the files an edit created or removed. Reverting an `asset-add` while the file is still in the `assets/` folder does not stick: the next folder sync re-registers it (folder contents win; delete the file too if you want the revert to hold). Reverting an `asset-rm` cannot bring back a deleted file; the restored registration itself gets pruned by the next folder sync (logged as `asset-prune`).
- Batch phrase cuts persist the phrase (`cut-text`), not the resolved word ids: if an external agent edits the transcript between render and save, the server can cut different words than the optimistic UI showed. The serialized save chain makes this unlikely; reload after external CLI edits (already the rule below).
- Reload the browser after CLI edits to see changes in the editor.
- A local `.openklip/projects-root` affects `projectsRoot()` for CLI and server started from that cwd (intentional; isolate tests with a clean temp cwd).
